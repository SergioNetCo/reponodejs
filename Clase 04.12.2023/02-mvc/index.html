1. Introducción

Poco despues de terminar con Hugo empezamos a ver el modelo vista controlador, un concepto muy importante que todo diseñador debe conocer. Es un patrón de diseño que separa la lógica de la aplicación en tres componentes: el modelo (datos y lógica), la vista (interfaz de usuario) y el controlador (gestiona la interacción entre modelo y vista).

3. Creación de MVC en JS

Esta práctica fue un poco más compleja, ya que recreamos a traves de JS lo que se basa el modelo. Sin tener mucha idea de lo que trataba y copiando codigo, logramos recrear el patron de manera bastante sencilla. No me voy a poner técnico pero si voy a explicar un poco los pasos que fuimos haciendo. El programa se basaba en un controlador de tareas, donde ponias la tarea que querías registrar y se añadia.

Pasos realizados:
  1. Estructura de el programa: debia tener el modelo (taskmodel.js), la vista (index.ejs y add.ejs) y por ultimo el controlador. (taskController.js)
  2. Creamos a través de JS un servidor para llevar la aplicación a un "servidor web" dentro de localhost.
  3. Taskmodel.js y index.ejs sirven para simular una base de datos y el index para hacer la página de inicio.
  4. Add.ejs era para el botón, donde cada vez que le dabas añadia una nueva tarea.
  5. Por último taskController.js que servia para registrar y añadir sin problema los taks creados.
