1. Laboratorios realizados:

Dimos un total de 5 tareas de GIT, subidas a github para ir realizandolo poco a poco. Antes que nada dimos la estructura y el conocimiento básico necesarios para empezar. Resumiendo muy rápido el contenido de cada práctica, se basó en:
  1. La primera práctica nos enseño a como crear respositorios, hacer cambios, ver el historial, navegar entre commits, etiquetarlos y ver diferencias de archivos.
  2. La segunda en como deshacer cambios en diferentes etapas del flujo de trabajo en diferentes ramas.
  3. Esta era sobre el archivo .gitignore para aprender a ignorar archivos y directorios especificos en un respositio. Tambien empezamos un poquillo con NodeJS, pero muy poco.
  4. Resolución de conflictos, como se producen y como resolverlos a la hora de trabajar en un equipo.
  5. Aprendimos a trabajar con ramas para desarrollar características o arreglar erroes en un ambiente aislado sin afectar a la rama principal.

Aquí dejo el respositorio donde se realizaron las 5 prácticas y tiene todo documentado: Repositorio

2. "Examen" de Laboratorios:

Despues de saber bastante sobre GIT realizamos un pequeño examen que se basaba en crear desde cero un nuevo flujo de trabajo donde había que hacer un fork de el repositorio de Alfonso para luego crear una rama y modificar ahí el archivo readme.md, luego, habría que envíarselo de manera que habría conflicto. En conclusión: realizar todo lo anterior pero en una hora y poder solucionar problemas y conflictos lo más rápido posible.

3. ¿Como ha afectado saber GIT?

Yo pensé que sería algo mas que se daría en clase, pero tiene un uso especial. Saber que en cualquier momento puedes subir tu código o incluso documentar algo en cuestión de segundos me parece que tiene un uso buenisimo. De hecho, en la empresa le estoy dando uso en repositorios privados para enseñar a empleados como usar diferentes sistemas, ahí les subo el código y demás y es bastante sencillo. Me parece muy revolucionario y muy bueno haberlo aprendido.
